## 시스템 테스트 방법
1. UI Testing / End-To-End Testing
    - 비용과 시간이 많이 들고 복잡해서 전체 비율의 10%정도를 구현하는 걸 권장
2. Integration Testing
    - 비용과 시간이 많이 들고 복잡해서 전체 비율의 20%정도를 구현하는 걸 권장
3. Unit Testing
    - 가장 쉽고 효과가 좋다. 전체 비율의 70%정도를 구현하는 걸 권장
