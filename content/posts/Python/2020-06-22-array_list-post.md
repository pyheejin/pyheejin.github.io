### Array(배열)

```js
// array 정의
int[] arraies = new int[6];

// array에 값 저장
arraies[0]=2;
arraies[1]=4;
arraies[2]=6;
arraies[3]=8;

// array의 길이 확인
// 처음 정의할 때 6으로 지정했기 때문에 4개의 값만 입력되어도 배열의 길이는 6이다.
System.out.println(arraies.length); // 6
```
- 다수의 데이터를 그룹핑해서 효율적으로 관리할 수 있는 데이터 스트럭쳐
- index가 있다.(값에 대한 유일한 식별자)
    - index를 이용한 조회가 빠르다.
- 요소를 삭제해도 그 공간은 남아있어서 메모리를 차지하게 된다.
    - 조건문으로 빈 요소를 제외할수는 있지만 조건문을 많이 사용하는 것은 좋지 않다.
- 지역성(cache hit)을 가지고 있어서 메모리 관리가 용이하다.
- 정의와 동시에 배열의 크기롤 정하며, 이후에 변경 불가하다.
    - 기존의 배열은 그대로 두고, 새로운 길이로 지정된 배열을 새로 지정(메모리가 있는지 탐색 필요하다.)
    - 기존 배열의 데이터 복사 후 기존 배열을 삭제
    - 총 3번의 작업 및 메모리 탐색이 필요하기 때문에 리소스 낭비가 크다.

### List(리스트)

```python
lists = [2, 4, 6, 8]
lists.append(10) # 리스트 뒤에 10 추가 [2, 4, 6, 8, 10]
lists.pop(2) # 인덱스 2의 값을 리턴 후 삭제 [2, 4, 8, 10]
```
- 순서가 있는 요소의 모임
- index가 없고 포인터로 다음 데이터의 위치를 가리키고 있어서 삽입, 삭제가 용이하다.
- 지역성이 없어서 배열보다 검색이 느리다.
- 대표기능(poeration)
    - 처음, 중간, 끝에 요소를 추가 및 삭제할 수 있다.
    - 리스트에 데이터가 있는지 확인하는 기능
    - 리스트의 모든 데이터에 접근할 수 있는 기능

## 차이점
- 배열 : 데이터의 크기가 변동되지 않고, 삽입 및 삭제가 일어나지 않으며 검색을 많이 사용할 때 유리하다.
- 리스트 : 데이터의 삽입, 삭제가 많고 검색이 적을 때 유리하다.
